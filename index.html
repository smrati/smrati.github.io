Explaining asynchronous programming in Node.js in easiest way possible

Published: Sun 09 January 2022
By Smrati Kumar Katiyar

In Web Development.

tags: node.js

Look at the synchrous code below

console.log('First Line');
console.log('Second Line');

Its output is: 

First Line
Second Line

Now look at the asynchronous code below 

setTimeout(() => console.log('First Line'), 1000);
console.log('Second Line');

Its output is: 

Second Line
First Line

Now if you compare both the codes you can identify in second block of code node gets and asynchronous function to execute setTimeout(() => console.log('First Line'), 1000); what this function mean is wait for 1000 milli seconds before executing code the anonymous function () => console.log('First Line')
So this anonymous function is the callback which will be executed after 1000 milliseconds.
But node being an asynchronous runtime don't wait for 1000 milli seconds instead it immediately move to second line for execution after sending first line to event loop.
